# 角色A 工作报告

## 职责

后端规则管线。接收 B 的结构化请求，经过标准化→检索→元数据提取→评分排序，返回完整菜谱列表。

```
B(LLM分析) → RecommendRequest → A.run_pipeline() → list[RecipeRecord] → B(继续)
```

---

## 输入

角色B 传来 `RecommendRequest`：

```json
{
  "ingredients_text": "西红柿、鸡蛋2个",
  "servings": 2,
  "time_limit_min": 20,
  "difficulty": "简单",
  "flavor": "不辣",
  "allergens": ["花生"],
  "excluded": [],
  "equipment": ["炒锅"]
}
```

---

## 输出

返回 `list[RecipeRecord]` 给角色B：

```json
[
  {
    "recipe_id": "R001",
    "title": "番茄炒蛋",
    "cuisine": "家常菜",
    "tags": ["快手菜", "下饭菜", "鸡蛋类"],
    "difficulty": "简单",
    "estimated_time_min": 15,
    "servings": 2,
    "core_ingredients": ["番茄", "鸡蛋"],
    "seasonings": ["食用油", "盐", "糖"],
    "optional_ingredients": ["葱"],
    "equipment": ["炒锅"],
    "allergens": ["鸡蛋"],
    "body": "番茄切块，鸡蛋打散...",
    "retrieval_score": 94.0
  }
]
```

- `retrieval_score` 为最终评分（0-100），已排序
- `body` 包含菜谱做法全文（RAGFlow 模式）
- 过敏原/忌口冲突的菜不返回（硬阻断）
- 缺核心食材越少越靠前，厨具匹配优先

---

## 内部流程

```
RecommendRequest
  │
  ├─ split_ingredients_text()   解析食材文本 → 食材名列表
  ├─ normalize_ingredients()    别名映射(144条) + 去重 + 分类 + 数量解析
  ├─ retriever.search_ids()    关键词匹配(stub) / 向量检索(RAGFlow) → [(id, score)]
  ├─ meta.get_or_create()       SQLite缓存 / LLM提取 + normalize → 结构化元数据
  ├─ build_feature()            硬过滤(过敏/忌口) + 软约束评分
  ├─ score_and_rank()           公式打分 + 排序
  ├─ body 补全                  从 retriever 获取全文
  │
  └─ list[RecipeRecord]
```

**评分公式：**
```
Base   = 45×核心覆盖率 + 15×检索分 + 10×口味 + 10×时间 + 5×难度 + 15×厨具比例
Penalty = 25×核心缺失比例 + (超时+10)
Final   = clamp(Base-Penalty, 0, 100)
```

**排序：** 缺核心少优先 → 厨具匹配优先 → 分高优先 → 检索分高优先

**硬阻断：** 过敏原冲突 / 忌口冲突 → 直接从结果移除

## 模块总览

| 模块 | 职责 |
|---|---|
| `rules/pipeline.py` | A 线统一入口 `run_pipeline()` |
| `rules/normalizer.py` | 食材别名映射(144条)、数量解析、文本分拆、分类 |
| `rules/staples.py` | 基础调料白名单(12项) |
| `rules/scorer.py` | 硬过滤(过敏/忌口)、难度符合度、厨具优先级、评分排序 |
| `retrieval/base.py` | `BaseRetriever` 抽象接口 |
| `retrieval/stub.py` | 12道种子菜谱，关键词精确匹配 |
| `retrieval/ragflow.py` | RAGFlow 向量检索 + search_ids + get_full_text + 自动降级 |
| `recipes/meta.py` | SQLite 缓存 + get_or_create(id, text) + normalize |
| `recipes/extractor.py` | LLM 提取菜谱元数据 + 规则兜底 |

## recipes 模块使用说明

### `RecipeMetaStore` — 元数据缓存管理器

```python
from recipes.meta import RecipeMetaStore
from recipes.extractor import llm_extract_recipe

# 创建实例（注入 extractor，自动处理缓存未命中）
meta = RecipeMetaStore(
    db_path="data/recipes.db",
    llm_extractor=llm_extract_recipe,   # 可选，不传则只读缓存
)
```

**接口：**

| 方法 | 说明 |
|---|---|
| `meta.get(id)` | 只查缓存，无数据返回 None |
| `meta.get_or_create(id, text)` | 查缓存 → 有则返回；无则调 extractor 提取 → 入库 → 返回 |
| `meta.put(id, record)` | 手动写入/更新 RecipeRecord（自动 normalize） |
| `meta.has(id)` | 检查是否已缓存 |
| `meta.batch_get_or_create([(id, text), ...])` | 批量获取/创建 |

**工作逻辑：**

```
get_or_create("肉蟹煲", "肉蟹煲的做法\n- 螃蟹 500g\n...")
         │
         ├─ SQLite 命中 → 直接返回 RecipeRecord
         │
         └─ 未命中 → llm_extract_recipe(text)
                       │
                       ├─ LLM 可用 → 提取 {title, core_ingredients, ...}
                       └─ LLM 不可用 → 规则从正文抓食材
                       │
                       ▼
                  normalize → 写入 SQLite → 返回 RecipeRecord
```

### `llm_extract_recipe` — LLM 菜谱提取器

```python
from recipes.extractor import llm_extract_recipe

# 独立调用
result = llm_extract_recipe("# 番茄炒蛋\n- 番茄 2个\n...")
# → {"title": "番茄炒蛋", "core_ingredients": ["番茄", "鸡蛋"], ...}

# 通常注入 meta 使用
meta = RecipeMetaStore(db_path="...", llm_extractor=llm_extract_recipe)
```

**内部逻辑：** 优先调 llm_client → 不可用则 urllib 直调 SiliconFlow → 失败则规则兜底（正则从正文抓食材行）。

### 典型用法

```python
# pipeline 中的实际调用
text = retriever.get_full_text("肉蟹煲")   # 从 RAGFlow 取全文
recipe = meta.get_or_create("肉蟹煲", text) # 提取元数据 + 入库
# → RecipeRecord(title="肉蟹煲", core_ingredients=["螃蟹","土豆"], difficulty="中等", ...)
```
