# 今晚吃什么 — 食材推荐菜谱

**7天 × 6小时 | 单人 | 前端 + LangGraph + RAGFlow**

```
用户输入食材 → 标准化 → 检索菜谱 → 匹配评分 → 生成做法 → 安全审查 → 菜谱卡片
```

---

## 一、目标

输入食材文本，12秒内返回1~3道可直接制作的菜，标注缺失食材、步骤、安全提醒。

| 核心原则 |
|---|
| 推荐必须可追溯知识库（不凭空捏造菜谱） |
| 过敏/忌口 = 硬拦截，口味/时间 = 软排序 |
| LLM 负责理解生成，规则代码负责计算排序 |
| Agent 之间传 JSON，不传大段自然语言 |

---


### Agent 分工

| Agent | 做什么 | 输出 |
|---|---|---|
| **ChefCoordinator** | 总控调度、整合输出 | `RecommendationResponse` |
| **RecipeMatcher** | 判断食材覆盖、偏好符合、冲突 | `CandidateFeature[]` |
| **CookingGuide** | 生成准备、步骤、火候、替代 | `{prep, steps, heat_tips}` |
| **FoodSafetyReviewer** | 过敏/忌口拦截、生食加热提醒 | `{passed, severity, issues}` |

### 确定性规则（不走LLM）

- 食材别名映射（50+ 组）
- 基础调料默认拥有（盐、油、生抽、葱姜蒜...）
- 评分公式硬计算
- JSON Schema 校验

---

## 三、评分公式

```
Base = 45×核心覆盖率 + 20×检索分 + 15×偏好 + 10×时间 + 10×厨具
Penalty = 25×核心缺失比例 + (超时?10:0)
最终分 = clamp(Base-Penalty, 0, 100)
```

排序优先级：**不阻断 → 核心缺失少 → 分高 → 时间短**

## 七、7天计划

| 天 | 重点 | 验收标准 |
|---|---|---|
| **1** | 环境+模板+20道种子 | 手工召回"番茄炒蛋" |
| **2** | 80~120道+别名表+检索 | Recall@10 ≥ 90% |
| **3** | LangGraph 8节点 | 端到端返回合法JSON |
| **4** | 评分+normalizer+API | 核心缺失不排第一 |
| **5** | 前端+卡片+动画+对接 | 页面完成真实请求 |
| **6** | 50条测试+安全专项 | 整体通过率 ≥ 85% |
| **7** | 修阻断+文档+彩排 | 连续3次演示成功 |

## 九、风险

| 风险 | 应对 |
|---|---|
| 模型JSON不稳定 | 低温度 + 规则兜底 |
| 菜谱质量不一致 | 模板校验+交叉抽查 |
| 延迟过高 | 候选≤10、关Rerank |
| 前端时间失控 | 降级纯HTML |
| 网络/服务故障 | 备录屏+固定JSON |
